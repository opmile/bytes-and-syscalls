## **Sistema que aguarda bytes chegarem da NIC (Network Interface Card)**

"Aguardar bytes chegarem da NIC" refere-se ao momento em que o sistema operacional ou a aplicação está pausado, esperando que os dados físicos percorram a rede e sejam processados pela placa de rede. No nível mais baixo, esse processo é regido pelo ciclo de I/O, onde a operação de Read fica suspensa até que os dados solicitados estejam disponíveis na memória para processamento.

Nesse fluxo, a NIC recebe os pacotes e, via DMA (Acesso Direto à Memória), entrega os bytes diretamente em buffers de recepção na memória RAM. Assim que os dados estão prontos, a placa gera uma interrupção para avisar à CPU que a operação de leitura pode ser concluída. O inverso ocorre no Write, quando o sistema envia dados para o buffer da NIC para transmissão física.

Se o sistema permanece nesse estado de espera, geralmente é sinal de lentidão na rede ou congestionamento no buffer de entrada. Além disso, a aplicação pode estar aguardando que o protocolo TCP preencha lacunas de pacotes perdidos ou reordene segmentos antes de entregar o fluxo completo para a camada superior.
